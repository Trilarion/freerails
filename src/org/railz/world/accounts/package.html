Provides classes to record transactions in a players account.

Each player has a {@link org.railz.world.accounts.BankAccount} that stores a growing list of {@link org.railz.world.accounts.Transaction Transactions}. Transactions are used represent bills and receipts for a player. Totals can be calculated by adding up the relevant transactions in the players account.

Transactions are categorised into their appropriate accounting category (operating expense, interest, tax etc.). This allows profit and loss accounts to be prepared, as well as the calculation of any applicable taxes.

Asset/Liability tracking

Each material asset represented by an object in the game world (trains, track, stations etc.) should have an associated book value. This value should take into account any depreciation. The depreciation can be calculated from the difference between book and purchase price. They should also instrument any current liabilities (e.g maintenance) that might be incurred.

Other assets, such as accounts, should be handled on a per class basis.
