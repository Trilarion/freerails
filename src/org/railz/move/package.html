Provides classes that encapsulate changes to the game world. The classes implement the GoF command pattern, and they are referred to as 'moves'. All moves should implement the interface {@link org.railz.move.Move}. The javadoc comment for the interface move sets out the contract they should obey in addition to implementing the methods it defines.

Guide lines for writing moves

(1) Implement the interface Move and follow the contract described in the javadoc for the interface.

(2) Rather than writing the class from scatch, it is probably better to extend one of the generic moves class.

if the move.. then extend
adds an item to a list {@link org.railz.move.AddItemToListMove}
removes an item from a list {@link org.railz.move.RemoveItemFromListMove}
changes an item in a list {@link org.railz.move.ChangeItemInListMove}
does several things {@link org.railz.move.CompositeMove}

(3) Consider writing a junit testcase for the move, and if you do, make the testcase a extend {@link org.railz.move.AbstractMoveTestCase}.
