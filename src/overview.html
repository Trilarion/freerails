Coding guidelines

Brief overview of the architecture

(1) The model and the view are separated. The classes that compose the model, referred to as the game world, are in the jfreerails.world.* packages. The view classes are in the jfreerails.client.* packages.

(2) All changes to the game world that occur after the game has started, referred to as moves, are done using the classes in the package jfreerails.move There are more details here.
