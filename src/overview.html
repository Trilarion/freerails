Brief overview of the architecture

(1) The model and the view are separated. The classes that make up the model, referred to as the game world, are in the jfreerails.world.* packages. The view classes are in the jfreerails.client.* packages.

(2) All changes to the game world that occur after the game has started, referred to as moves, are done using the classes in the package jfreerails.move There are more details here.

(3) The client and server are separate. The game works as follows. When the game starts or a game is loaded, the server sends the client a copy of the game world. When the player does something, e.g. build track, the client sends a move to the server. The server validates the move and updates its copy of the game world, then sends the move back to the client (and to the clients belonging to any other players). The clients then, and only then, update their copies of the game world. Moves that are not caused by the player doing something, e.g. new cargo being generated at stations, are generated by the server.
